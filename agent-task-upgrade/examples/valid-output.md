# Контекст

Механизм маскирования получает запрос, заявленный как `application/json`, но тело запроса может не являться валидным JSON.

# Цель

Исключить сохранение или вывод исходного тела запроса, заявленного как JSON, если тело не является валидным JSON, сохранив текущее поведение для валидного JSON.

# Критерии приёмки

```gherkin
Scenario: Полное маскирование невалидного JSON-тела
  Given запрос содержит Content-Type: application/json
  When тело запроса не является валидным JSON
  Then при маскировании всё тело запроса заменяется маской
```

# Пользовательские ограничения

## Обязательные

- Не менять текущее поведение для валидного JSON.

## Предпочтительные

- Использовать существующий механизм маскирования.

## Запрещено

- Добавлять новую внешнюю зависимость.
